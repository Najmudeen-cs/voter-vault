How VoterVault Works
Security architecture & technical specification
Registration Flow
Step 1Voter enters personal details: name, DOB, gender, ID, constituency.
Step 2Details are serialized to JSON and encoded as UTF-8 bytes.
Step 3Payload is encrypted with AES-256-CBC using a user-defined password via PBKDF2 key derivation.
Step 4Encrypted bytes are hidden in the carrier image using LSB (Least Significant Bit) steganography — 1 bit per RGB channel.
Step 5The resulting PNG looks identical to the original photo but carries the hidden voter data inside.
Encryption Specification
AlgorithmAES-256-CBC via Web Crypto API
Key DerivationPBKDF2-SHA256 · 100,000 iterations · 32-byte key
Salt / IV16 bytes each, cryptographically random per session
Wire Format[4B magic 0xDEADBEEF][16B salt][16B IV][ciphertext]
Header[4B payload length][1B marker 0xAA] at LSB bit offset 0
LSB Steganography
MethodLeast Significant Bit substitution in RGB pixel channels.
Capacity3 bits per pixel (1 per R, G, B channel).
CarrierAny PNG / JPG / BMP / WEBP image uploaded by the user.
VisibilityCompletely invisible to the human eye. No perceptible change.
Best PracticeUse PNG. JPG lossy compression will destroy embedded data.
Privacy & Security
Processing100% client-side in the browser. No data is ever sent to a server.
StorageNo cookies, no localStorage, no telemetry of any kind.
PasswordPassword never leaves the browser. Never stored anywhere.
Voter IDAuto-generated unique ID per registration (random + timestamp).
TamperAny image manipulation destroys the embedded data, acting as tamper detection.
Engine Web Crypto API
Steg Method Image LSB
Encryption AES-256-CBC
Network Isolated · Local Only
Database Not configured
